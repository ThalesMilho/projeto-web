possíveis perguntas:

1. "Deu erro de CORS!"

O que é: O navegador bloqueou o React de ler dados do Django por segurança.

Solução: Confirme se a URL que o React está rodando (ex: localhost:3000) está exatamente igual na lista CORS_ALLOWED_ORIGINS do seu settings.py.

2. "Qual é a URL base?"

Resposta: "Enquanto estamos rodando localmente, todas as suas requisições fetch ou axios devem bater em: http://127.0.0.1:8000/api/...".

3. "O JSON vem em snake_case ou camelCase?"

O Problema: Python usa nome_usuario (snake_case). JavaScript usa nomeUsuario (camelCase).

Aviso: "Por padrão, o Django vai te mandar nome_usuario. Se você quiser converter no front, tudo bem, ou podemos instalar um parser no back depois. Por enquanto, espere snake_case".

4. "Como eu sei o que mandar pro Back?"

Aviso: "O Django espera JSON. Não mande 'Form Data' ou XML. Seus Headers no React devem ter Content-Type: application/json".

5. "E as imagens?"

Aviso: "Em ambiente de desenvolvimento (localhost), o Django serve as imagens. Em produção a estratégia muda (AWS S3, etc). Por enquanto, vou te passar URLs completas (com http://localhost...)."
